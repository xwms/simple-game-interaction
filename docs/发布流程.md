# 发布流程

## 前置要求

- Git 已配置，可推送至 GitHub
- 有 GitHub 仓库的写入权限

## 步骤

### 1. 更新版本号

编辑 `package.json`，修改 `version` 字段：

```json
{
  "version": "0.0.4"
}
```

### 2. 更新 CHANGELOG.md

在 `CHANGELOG.md` 顶部添加新版本记录：

```markdown
## [0.0.4] — 2026-06-22

### 新增
- 新功能描述

### 修复
- 修复内容
```

格式参考已有的历史记录。

### 3. 提交并打 tag

```bash
# 先推送到 main（触发 CI test）
git add -A
git commit -m "chore: bump version to 0.0.5"
git push origin main

# 打 tag 并推送（触发 CI build + release）
git tag v0.0.5
git push origin v0.0.5
```

### 4. 等待 CI 完成

GitHub Actions 自动执行（仅推送 tag 时触发 build + release）：

| 阶段 | 说明 |
|------|------|
| test | 三平台构建 + 单元测试 |
| build | 打包安装程序（.exe / .dmg / .AppImage / .deb） |
| create-release | 创建 GitHub Release，上传产物 |

### 5. 验证

- 打开 https://github.com/xwms/simple-game-interaction/releases
- 确认新版本已创建，包含以下安装包：

| 平台 | 文件 |
|------|------|
| Windows | `SGI-Setup-0.0.4-win.exe` |
| macOS (Intel) | `SGI-0.0.4-mac.dmg` |
| macOS (Apple Silicon) | 暂需本地构建或 Rosetta 2 运行 x64 版本 |
| Linux | `SGI-0.0.4-linux.AppImage` |
| Linux | `SGI-0.0.5-linux.deb` |

## 注意事项

- 先推送 commit 到 main（触发 test），再单独推送 tag（触发 build + release）
- tag 名必须为 `v` 开头，与 `package.json` 中的版本号保持一致
- 推送 tag 后 CI 自动执行，无需手动操作
- macOS 安装包无开发者签名，首次运行时需要在"系统设置 → 隐私与安全性"中允许打开
