# 发布流程

## 前置要求

- Git 已配置，可推送至 GitHub
- 有 GitHub 仓库的写入权限

## 步骤

### 1. 更新版本号

编辑 `package.json`，修改 `version` 字段：

```json
{
  "version": "X.Y.Z"
}
```

### 2. 更新 CHANGELOG.md

在 `CHANGELOG.md` 顶部添加新版本记录，参考已有格式：

```markdown
## [X.Y.Z] — YYYY-MM-DD

### 新增
- 新功能描述

### 修复
- 修复内容
```

### 3. 提交并打 tag

```bash
git add -A
git commit -m "chore: bump version to X.Y.Z"
git tag vX.Y.Z
git push origin main --tags
```

> 如果只想先跑 test 确认通过再打 tag，可以分两步：
> ```
> git push origin main          # 仅触发 test
> git push origin vX.Y.Z        # 触发 build + release
> ```

### 4. 等待 CI 完成

GitHub Actions 自动执行：

| 阶段 | 触发条件 | 说明 |
|------|----------|------|
| test | push/PR to main | 三平台构建 + 单元测试 |
| build | 推送 tag v* | 打包安装程序（.exe / .dmg / .AppImage / .deb） |
| create-release | build 完成后 | 创建 GitHub Release，上传产物 |

### 5. 验证

- 打开 https://github.com/xwms/simple-game-interaction/releases
- 确认新版本已创建，包含以下安装包：

| 平台 | 文件 |
|------|------|
| Windows | `SGI-Setup-X.Y.Z-win.exe` |
| macOS (Intel) | `SGI-X.Y.Z-mac-x64.dmg` |
| macOS (Apple Silicon) | `SGI-X.Y.Z-mac-arm64.dmg` |
| Linux | `SGI-X.Y.Z-linux.AppImage` |
| Linux | `SGI-X.Y.Z-linux.deb` |

## 注意事项

- tag 名必须为 `v` 开头，与 `package.json` 中的版本号保持一致
- 推送 tag 后 CI 自动执行，无需手动触发
- build 和 create-release 只在推送 tag 时运行，普通推送仅跑 test
- macOS 双架构（x64 + arm64）在 macos-latest (ARM64) runner 上同时构建
- macOS 安装包无开发者签名，首次运行时需在"系统设置 → 隐私与安全性"中允许打开
- 如推送 tag 后需要重新发布，先删除旧 tag 再推送新的：
  ```bash
  git tag -d vX.Y.Z
  git push origin --delete vX.Y.Z
  git tag vX.Y.Z
  git push origin vX.Y.Z
  ```
